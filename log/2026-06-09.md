## 一、修复用户注册 Bug

### 1.1 密码不能为空

**根因：** `Staff.java` 中 `@JsonIgnore` 注解同时阻止序列化和反序列化。前端传来的 `password` 字段被 Jackson 直接丢弃，后端 `staff.getPassword()` 始终为 `null`，触发 `ValidationUtil.requireNotBlank(null, "密码")` → "密码不能为空"。

**解决：** 替换为 `@JsonProperty(access = JsonProperty.Access.WRITE_ONLY)`，允许写入（反序列化），禁止读出（序列化），既解决注册问题又不泄露密码哈希。

**修改文件：** `backend/.../entity/Staff.java:6,23-24`

### 1.2 注册失败后表单被清空

**根因：** `RegisterPage.vue` 中表单重置代码写在 `finally` 块中，无论成功失败都清空，用户需从头重填。

**解决：** 将表单清空逻辑移到 `try` 块内（`ElMessage.success` 之后），仅在注册成功时清空。失败时保留已填数据。

**修改文件：** `frontend/src/views/RegisterPage.vue:78-91`

### 1.3 电话号码校验提示

**根因：** `12345678901` 是 11 位数字但第二位是 `2`，不匹配正则 `^1[3-9]\d{9}$`。后端错误提示"手机号必须为11位手机号码"有误导性。

**解决：** 后端提示改为"请输入正确的11位手机号码"。

**修改文件：** `backend/.../util/ValidationUtil.java:32`

### 1.4 重复用户名 → 500 错误

**根因：** `username` 字段有 UNIQUE 约束，重复注册时 MySQL 抛出 `DuplicateKeyException`（`RuntimeException`），被兜底处理器返回 500 "服务器内部错误"。

**解决：**
- `StaffService.add()` 插入前先 `findByUsername` 查重，存在则抛 `BusinessException("用户名已存在，请换一个")`
- `GlobalExceptionHandler` 新增 `DataIntegrityViolationException` 专门处理，防止其他约束冲突也返回 500

**修改文件：** `backend/.../service/StaffService.java:42-46`、`backend/.../common/GlobalExceptionHandler.java:13-14,32-41`

## 十、全系统三层次数据校验

### 第一层：前端实时过滤（输入即拦截）

| 字段类型 | 涉及页面 | 过滤方式 |
|---------|---------|---------|
| 手机号 | CustomerPage / StaffPage / RegisterPage | `maxlength="11"` + `@input` 自动去除所有非数字字符 |
| 金额 | BusinessPage | `el-input-number` 原生禁止字母，`:precision="2"` 限制两位小数 |

### 第二层：前端提交校验（保存前验证）

| 页面 | 校验内容 |
|------|---------|
| CustomerPage | 手机号 11 位纯数字 + 以 1 开头第二位 3-9；邮箱正则格式 |
| BusinessPage | 金额必须 > 0；金额最多两位小数 |
| StaffPage | 手机号 11 位合法格式 |
| RegisterPage | 手机号 11 位合法格式 |
| WorkOrderPage | 客户/业务/区域必填 |

### 第三层：后端校验（最后防线）

`ValidationUtil` 新增方法：

| 方法 | 功能 |
|------|------|
| `isDigitsOnly(str)` | 字符串是否全为数字 |
| `requireDigitsOnly(value, fieldName)` | 非纯数字 → 抛异常 |
| `requireValidAmount(BigDecimal, fieldName)` | ≤0 或超过两位小数 → 抛异常 |
| `isPositiveInteger(str)` | 是否正整数 |

应用到三个 Service：
- `CustomerService.validateCustomer()`：手机号先校验纯数字 + 11位长度，再校验号段
- `BusinessService.validateBusiness()`：使用 `requireValidAmount` 替代简单 `>0`
- `StaffService.validateStaff()`：手机号与客户同等级别校验

**修改文件：** `backend/.../util/ValidationUtil.java`（增强）、`CustomerService.java`、`BusinessService.java`、`StaffService.java`、`CustomerPage.vue`、`BusinessPage.vue`、`StaffPage.vue`、`RegisterPage.vue`

## 十一、遇到的问题及解决

### 问题 1：Maven wrapper JAR 损坏

命令行 `mvnw compile` 报 `没有主清单属性`，无法通过命令行构建后端。

**解决：** 在 IntelliJ IDEA 中直接 Build / 重启 Spring Boot。

### 问题 2：`selectCount` 返回类型不匹配

`SeaPoolService.countTodayClaims()` 中 `customerMapper.selectCount(wrapper)` 返回 `Long`，无法直接 `(int)` 强转。

**解决：** 改为 `.intValue()`。

### 问题 3：`BusinessService.listByOwnerIds()` 未定义

`BusinessController.list()` 调用了不存在的 `businessService.listByOwnerIds()`。

**解决：** 在 `BusinessService` 中新增该方法，按 `ownerIds` 列表过滤。

### 问题 4：`WorkOrderService.getById()` 未定义

`WorkOrderController` 权限校验需要根据 ID 查询工单。

**解决：** 在 `WorkOrderService` 中新增 `getById(Integer id)` 方法。
