## 三、修复仪表盘 500 错误

### 根因

数据库 `crm_system` 中缺少 `follow_up` 表。迁移脚本 `migration_v2.sql` 定义了该表但从未执行。Dashboard API 调用 `followUpService.countToday()` 时，MyBatis 查询不存在的表 → `RuntimeException` → `GlobalExceptionHandler` 返回 HTTP 500 + "服务器内部错误"。

### 解决

手动执行 `CREATE TABLE follow_up (...)`，表创建成功后仪表盘正常工作。

**涉及 SQL：**
```sql
CREATE TABLE IF NOT EXISTS follow_up (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '跟进记录ID',
    customer_id INT NOT NULL COMMENT '客户ID',
    content TEXT NOT NULL COMMENT '跟进内容',
    staff_id INT COMMENT '跟进人ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (customer_id) REFERENCES customer(id) ON DELETE CASCADE
) COMMENT='客户跟进记录表';
```

## 四、修复 VIP 颜色未生效

### 根因

数据库 `customer` 表中存储的是旧版 VIP 值（`VIP会员`、`金卡会员`、`钻石会员`），来自 `seed_data.sql` 旧数据。前端下拉框选项是 `VIP1`/`VIP2`/`VIP3`，`vipTagClass()` 函数无法匹配旧值，全部落入默认灰色 `vip-normal`。

### 解决

1. 数据库 `UPDATE customer SET vip_level = 'VIP1' WHERE vip_level = 'VIP会员'`（同理 VIP2/金卡会员、VIP3/钻石会员）
2. `seed_data.sql` 中全部替换为新值
3. `vipTagClass()` 增加旧值兼容映射

## 九、遇到的问题及解决

### 问题 1：VIP 筛选后分页 total 不变，列表为空

**根因：** 初版 VIP 筛选仅在前端客户端过滤 `tableData`（当前页数据），分页 `total` 始终取后端全量总数。跨页场景下当前页可能无匹配记录。

**解决：** VIP 筛选下沉到后端 `CustomerService.search()`，通过 `wrapper.eq(Customer::getVipLevel, vipLevel)` 在数据库层过滤，分页统计精确一致。

### 问题 2：VIP 筛选点击无效

**根因：** `el-radio-group` 的 `@change` 事件可能早于 `v-model` 更新触发，导致 `fetchData()` 读取到旧的 `vipFilter.value`。

**解决：** 移除 `@change`，改用 Vue `watch(vipFilter, ...)` 监听，确保 ref 更新后再发请求。

### 问题 3：Maven wrapper JAR 损坏

命令行 `mvnw compile` 报 `没有主清单属性`，无法通过命令行构建后端。改为在 IntelliJ IDEA 中直接运行 Spring Boot。
